# PR 作成スキル

## Instructions

### 1. 背景のヒアリング

PRを作成する前に、**なぜこのPRを作ったか**をユーザーに必ず確認する。

- コードの差分だけでは読み取れない背景や動機を聞き出す
- 例: 「このPRを作った背景を教えてください」「なぜこの変更が必要になりましたか？」
- ユーザーが既に背景を伝えている場合はスキップしてよい

### 2. 差分の把握

以下を実行して変更内容を把握する：

```bash
git log --oneline main..HEAD
git diff main...HEAD
```

### 3. description の作成方針

- **冒頭に、なぜこのPRを作ったかの背景を書く。** コードからは読み取れない動機や経緯を記載する
- **差分をそのまま解説するだけのdescriptionにしない。** 差分はコードを見ればわかる。descriptionにはコードからわからない情報を書く
- PRが複雑な場合は、レビュワーの負担を減らすためにざっくりとした説明を加えてよい。ただしそれは差分の羅列ではなく、変更の意図や全体像の説明であること
- **差分を生成する際にコマンドを使った場合（コード生成、マイグレーション、フォーマッタなど）は、どのようなコマンドを使ったかをdescriptionに記載する。** レビュワーが再現・検証できるようにするため
- Test plan には、CIで既に担保されている検証項目は記載しない

### 4. PR の作成

```bash
gh pr create --title "<タイトル>" --body "<description>"
```

- タイトルは日本語で書く
- タイトルは70文字以内
- ブランチがリモートにpushされていない場合は先にpushする
